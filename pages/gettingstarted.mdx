## Wrist Overlay
The Wrist Overlay can be found on the back of your left hand by default. 

![Wrist Menu ](img/WristMenu.png)



> ### Enable/Disable Layout Mode
>Toggles [Layout Mode](gettingstarted#layout-mode) on or off without the need to use the controller binding.

>### Show/Hide Media Controls
>Allows you to control media on your device. Pressing this will replace the tracker battery indicators with media buttons. Pressing it again will return the tracker battery indicators. These controls will work on any media on the system. Pressing this will replace the tracker battery indicators with media buttons. Pressing it again will return the tracker battery indicators. >### Show/Hide System Stats
>Displays information about your system's resources. Clicking on this panel will open the system stats window.
>![System Stats](img/SystemStats.png)

>### Battery Life
>Displays battery life information in the order of Left Controller, Right Controller, Trackers in order of connection. Clicking on this panel will open the estimated battery life each device.
>![Battery Life](img/BatteryLevels.png)

## Layout Mode
<Callout type="info">
[Layout Mode](GettingStarted?id=layout-mode) can be opened by pressing the [Layout Mode](GettingStarted?id=layout-mode) button on the wrist overlay, or by pressing the correct binding to open it. Layout mode allows you to move overlays around, apply modifiers to overlays, change overlay capture targets, and change overlays settings.
Below are all of the default bindings and interaction methods for each controller. 
    
    _Note: You can change all bindings by opening the settings and clicking the "Bindings" button. This will take you to the SteamVR bindings page for XSOverlay._
</Callout>

![Layout Mode](img/LayoutMode.png)

> ### Add Window
> Adds a window to the overlay. 

>### Recenter Layout
> Re-centers layout to be in the same relative position and rotation automatically when opening [Layout Mode](gettingstarted#layout-mode). This is just like opening your overlay menu, but this is a manual menu.

>### Keyboard
> Enables/Disables the keyboard. 

>### Grid View
> Moves windows in a grid formation for quick organization.

>### Saved Layouts
> Gives users to access the saved layouts menu. This menu allows users to save, load, and delete layouts. 

>### Settings
> Opens up the setting menu for configuring XSOverlay options. See [Settings](settings) for more information.

>### Remove All Windows
> Removes all currently displayed windows. This requires two button presses for confirmation.
